I will show it to you again, using "*" to mark where trunk items will go and "$" to mark where glove box items will go.

With the inventory, we always want to make sure that the ascii's formatting is maintained. the items will be inserted into their respective spaces, but they won't take up horizontal character space. Does that make sense?

--------------------------------------------
|        TRUNK:             GLOVE BOX:     |
|[*                  ][$                  ]|
|[*                  ][$                  ]|
|[*                  ][$                  ]|

So you can see that items will be displayed from left to right, within these "[""]" sections.

Now, items that fit in the trunk section of the inventory include:
"Jerry Can" (adds between 10%-15% gas when clicked)
"Car Parts" (adds between 10%-15% engine when clicked)
"Brake Pads" (adds between 20$-30% brakes when clicked)
"Spare Tire" (adds 25% wheels when clicked)

Items also include a "drop" feature, which is brown, whereas the item text itself is red, indicating it can be clicked to use.

When an item is clicked/used, it disappears from the inventory.

When an item is dropped, it disappears from the inventory.


Let's now turn our attention to the CB radio. We need to make sure the button works. Remember, it's only a button in name. It should be clickable ascii. The clickable part will be: "|"10-2!"|"

Remember that this is only clickable when "Do you read me?" appears.

"Do you read me?" will appear based on a randomly generated timer when the game loads. It will select a time between 1 and 2 minutes. At the selected time, "Do you read me?" will appear for 10 seconds. For that 10 second window, the previously mentioned ascii button becomes clickable. When clicked, a second message appears, replacing "Do you read me?" It says, "There's a closer location!"
This second piece of text coincides with between 5km-20km removed from the 'til next stop km counter. After 10 seconds, the second message disappears.

Please note that when the FIRST message disappears, the randomizer selects a new future time at which the message will re-appear.
